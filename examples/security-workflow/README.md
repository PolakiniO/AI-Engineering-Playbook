# Security Workflow Example

This directory shows how the generic governance framework can be adapted to a security-focused workflow repository.

What stays generic:

- mandatory automatic skill usage
- strict-mode review behavior
- playbook-driven skill selection
- layered review model

What becomes repository-specific:

- security review posture and operating boundaries
- workflow-specific pipeline constraints
- repository-specific module ownership and interface boundaries
- persisted artifact conventions
- domain-specific skills such as discovery analysis, output triage, and validation review

Files here are examples of specialization, not the generic framework itself.
